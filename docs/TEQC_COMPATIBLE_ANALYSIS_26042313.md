# 26042313 QC comparison: teqc-compatible subset vs CEQC modern-only

## Important comparison boundary

The uploaded teqc report is generated by `teqc 2019Feb25`, while the CEQC report is generated by `ceqc 0.0.1`.  teqc is an old legacy tool and is not a reference for modern BDS-3, Galileo, QZSS, RINEX3/4 typed NAV, or multi-frequency MP combinations.  Compare only the legacy subset that both tools can represent.

The uploaded reports also show different calendar dates in their windows: teqc reports 2026-06-12 13:00:30 to 14:00:15, while CEQC reports 2026-06-05 13:00:30 to 14:00:15.  Numeric equality is therefore not a valid byte-level or full-summary comparison until the exact same OBS/NAV pair and time window are used.

## teqc-compatible common subset

These are suitable for trend/regression comparison when the same legacy OBS/NAV pair is used:

- GPS/GLONASS L1/L2 timelines for common satellites.
- Observation interval and epoch count.
- Legacy MP12/MP21.
- Mean S1/S2.
- Basic GPS unhealthy-SV and missing OBS/NAV lists.
- Position as a sanity check, not as a strict equality target if teqc rejects NAV records.

## CEQC modern-only section

These must be analyzed independently rather than compared to teqc:

- BeiDou MP26, MP27, MP62, MP67, MP72, MP76.
- GPS/QZSS multi-frequency MP15/MP25/MP51/MP52.
- Galileo/BDS/QZSS and RINEX3/4-specific outputs.
- S5/S6/S7/S8 statistics.
- RTCM3 MSM native decoding behavior.

## Implementation action in CEQC 0.0.1

The text report now labels:

- `teqc-compatible legacy QC subset`
- `CEQC modern-only multipath combinations`

This prevents BDS-3 and other modern diagnostics from being incorrectly treated as teqc regressions.
